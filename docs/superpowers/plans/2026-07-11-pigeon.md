# Pigeon Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build and publish an approval-gated Codex delegation plugin with an embedded inbox and native confirmation.

**Architecture:** A TypeScript workspace separates a signed delegation protocol, relay, local gateway, Codex adapter, MCP/app surface, and React UI. Integration tests connect two gateways to an in-process relay and use a deterministic fake Codex adapter; the production adapter speaks Codex app-server JSON-RPC.

**Tech Stack:** Node.js 22+, TypeScript, pnpm, Vitest, Zod, Fastify, WebSocket, MCP TypeScript SDK, React, Vite, Ed25519 via Node crypto.

## Global Constraints

- The embedded inbox is the primary approval surface.
- Approval requires a separate native confirmation showing sender, objective, workspace, and effective scope.
- Scope defaults to `discuss_only` and may be narrowed but never widened.
- The relay never receives Codex credentials or private identity keys.
- Production approval is unavailable outside the embedded app plus native confirmation flow.
- The public repository must contain no credentials or private environment data.
